| DEC-33 | 2026-07-02 | Trust Cloudflare's forwarded scheme/IP headers | `Program.cs` runs `app.UseForwardedHeaders(ForwardedHeaders.XForwardedFor \| ForwardedHeaders.XForwardedProto)` as the **first** middleware, with `KnownNetworks`/`KnownProxies` **cleared**. | The first prod deploy (`https://exec-plan.codexkw.co`) sits behind Cloudflare — TLS terminates at the edge, so the origin host receives plain HTTP. Without this, the app read `Request.Scheme==http` and (verified live) emitted `http://` redirect `Location`s on `/admin`, dropped `Secure` from the `SameAsRequest` antiforgery cookie, and would generate `http://` absolute URLs for any future email/reset/activation links. Placed first so redirects, cookies, and auth all see the corrected scheme. Clearing `KnownNetworks`/`KnownProxies` is required because Cloudflare's egress IPs are dynamic — but it means the headers are trusted from **any** upstream, so it is safe **only** while the origin host admits inbound traffic solely from Cloudflare (see the origin-firewall Open note). The admin auth cookie was already `CookieSecurePolicy.Always`, so it was never affected. |

- **DEPLOY gotcha (Docker/Linux)** `KuwaitShiftCalculator` resolves the Asia/Kuwait timezone from the OS tz database. A container built with `DOTNET_SYSTEM_GLOBALIZATION_INVARIANT=true` or without `tzdata` (e.g. Alpine) will throw `TypeInitializationException` on first use. The future Dockerfile MUST install `tzdata` (and not set invariant globalization), or add a fixed UTC+3 fallback. Not reachable on the current Windows dev target.
- **Origin must be firewalled to Cloudflare (DEC-33 precondition)** `UseForwardedHeaders` clears `KnownNetworks`/`KnownProxies`, so `X-Forwarded-Proto`/`-For` are trusted from any upstream. This is only safe while the origin host accepts inbound HTTP(S) **solely from Cloudflare's IP ranges** (IIS IP-and-Domain Restrictions or the host firewall). Until that is enforced, a request sent directly to the origin can spoof the scheme and client IP. Verify after the redeploy that a direct-to-origin hit is refused.
- **Multi-tenant scoping** Manager/Admin authority over execution ops (dashboard/close/broadcast/escalate/set-substitute) is plan/tenant-global in the spine (DEC-17, internally consistent across REST+hub+services). Add org/tenancy scoping before more than one organization goes to production.
